# LLM Roadmap

This folder is the working space for my transition into `LLM Application Engineering` and `AI Backend Engineering`.

The goal is not to study LLMs only as theory. The goal is to build strong, interview-ready, project-backed skills that match my current strengths in:

- Python
- FastAPI
- Backend development
- DevOps
- Cloud
- System design

This roadmap will grow over time. As I learn, I will keep adding:

- notes
- revision docs
- project ideas
- code experiments
- interview questions
- architecture patterns

## Main Direction

Primary target:

- `AI Backend Engineer with LLM specialization`

Secondary target:

- `LLMOps / AI Platform Engineer`

Why this path:

- It builds on my existing backend and DevOps foundation.
- It is more practical and job-oriented than jumping directly into pure ML research.
- It allows me to build visible projects quickly.
- It gives me a strong profile for real-world AI product teams.

## Learning Strategy

I will learn in this order:

1. LLM fundamentals for builders
2. RAG systems
3. Agents and workflows
4. LLMOps and deployment
5. Open-source model serving and local experimentation

Rule:

- Learn just enough theory to build.
- Build projects after every phase.
- Keep notes short, reusable, and revision-friendly.

## Weekly Study Pattern

Each week should follow this structure:

1. Learn one focused topic
2. Build one small feature
3. Write short notes
4. Save code examples
5. Write revision questions
6. Reflect on what is still unclear

Recommended weekly split:

- `5 days`: learning + building
- `1 day`: notes + revision
- `1 day`: recap + cleanup + GitHub update

## What To Avoid

- learning too many frameworks at once
- starting with heavy fine-tuning
- spending too much time on theory without projects
- using abstractions without understanding the basics
- building demos without evaluation or reliability thinking
