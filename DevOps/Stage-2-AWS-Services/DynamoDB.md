### **📌 Amazon DynamoDB - Graphical Summary**  

#### **🌍 Overview of DynamoDB**  
- **Fully managed NoSQL database** service.  
- Supports **key-value & document-based** storage.  
- **Auto-scales** to handle millions of requests per second.  
- **Low latency (<10ms)** for high-speed applications.  

---

#### **⚡ Key Features of DynamoDB**  
| Feature | Description |
|---------|------------|
| **Key-Value Store** | Stores data as key-value pairs. |
| **Auto Scaling** | Adjusts capacity based on traffic. |
| **On-Demand Mode** | Pay-per-request pricing (no provisioning needed). |
| **DynamoDB Streams** | Captures real-time changes in data. |
| **Global Tables** | Multi-region replication for global apps. |
| **DAX (DynamoDB Accelerator)** | Caching layer for ultra-low latency. |
| **TTL (Time-To-Live)** | Automatically deletes expired data. |

---

#### **📦 DynamoDB Storage Models**  
| Storage Model | Best For |
|--------------|---------|
| **Key-Value Store** | Fast lookups using unique keys. |
| **Document Store** | Stores JSON-like documents, flexible schema. |

---

#### **🔒 Security & Access Control**  
- **IAM Roles & Policies** → Manage access control.  
- **VPC Endpoints** → Securely access DynamoDB from VPC.  
- **Encryption (KMS)** → Encrypts data at rest & in transit.  

---

#### **💰 Pricing & Cost Optimization**  
| Pricing Model | Description |
|--------------|------------|
| **Provisioned Mode** | Pre-allocate read/write capacity. |
| **On-Demand Mode** | Pay only for actual requests. |
| **DAX Caching** | Reduces costs by caching queries. |

---

#### **⚖️ Performance & High Availability**  
- **Multi-AZ Replication** → Ensures durability.  
- **Global Tables** → Replicates data across AWS regions.  
- **DynamoDB Streams** → Captures changes in real time.
