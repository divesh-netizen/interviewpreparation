### **📌 Amazon S3 - Graphical Summary**

#### **🌍 Overview of S3**  
- Object storage for the cloud.  
- Scalable, durable, and secure.  
- Store any type of data (files, backups, media).  
- Highly available across multiple regions.  

---

#### **🗂️ Key Concepts in S3**  
- **Buckets**: Containers for organizing data.  
- **Objects**: The actual data/files stored in buckets.  
- **Storage Classes**:  
  - **Standard**: Frequently accessed data.  
  - **Infrequent Access**: Less frequently accessed data.  
  - **Glacier**: Archived data.  

---

#### **📦 Storage Classes**  
| Storage Class | Use Case | Durability |
|---------------|----------|------------|
| **Standard** | Frequently accessed | 99.999999999% |
| **IA (Infrequent Access)** | Backup, archives | 99.999999999% |
| **One Zone-IA** | Single AZ storage | 99.999999999% |
| **Glacier** | Archival storage | 99.999999999% |
| **Glacier Deep Archive** | Long-term archival | 99.999999999% |

---

#### **🔒 Security & Access Control**  
- **Bucket Policies**: Set permissions for who can access the data.  
- **IAM (Identity & Access Management)**: Controls user access.  
- **ACL (Access Control Lists)**: Fine-grained control over data access.  
- **Versioning**: Keeps multiple versions of an object.  
- **Encryption**: Data encryption at rest and in transit.  

---

#### **⚙️ S3 Features**  
- **Lifecycle Policies**: Automatically move data between storage classes.  
- **Event Notifications**: Trigger actions based on changes to data (e.g., Lambda, SQS).  
- **Cross-Region Replication**: Replicate data across different AWS regions.  

---

#### **📅 Data Management**  
- **Versioning**: Keeps multiple versions of an object.  
- **Object Locking**: Prevents deletion or modification of objects for compliance.  
- **Data Transfer Acceleration**: Speed up uploads using edge locations.  

---

#### **💰 Pricing & Cost Management**  
- **Pay-as-you-go**: Pay based on storage used, data transfer, and requests.  
- **Request Costs**: Per GET, PUT, DELETE request.  
- **Data Transfer**: Charges for transferring data out of S3.

### **🔹 Amazon S3 - Q&A Session**  

---

### **1️⃣ S3 Basics**  
1️⃣ **Q:** What is Amazon S3?  
   **A:** Amazon Simple Storage Service (S3) is an **object storage** service that allows you to store and retrieve any amount of data from anywhere.  

2️⃣ **Q:** What is the difference between object storage and block storage?  
   **A:**  
   - **Object Storage (S3)** – Stores data as objects with metadata and unique keys (best for backups, images, logs, and media).  
   - **Block Storage (EBS, Instance Store)** – Stores data in blocks like traditional hard drives (best for databases and OS storage).  

3️⃣ **Q:** What is an S3 bucket?  
   **A:** An **S3 bucket** is a container that stores objects (files, data, etc.). Each bucket has a **unique name** globally across AWS.  

4️⃣ **Q:** How many buckets can you create per AWS account?  
   **A:** By default, **100 buckets per account** (can be increased with a request).  

5️⃣ **Q:** How does Amazon S3 provide high availability?  
   **A:**  
   - Data is automatically replicated across multiple Availability Zones (AZs).  
   - Designed for **99.999999999% (11 nines) durability**.  

---

### **2️⃣ S3 Storage Classes**  
6️⃣ **Q:** What are the different S3 storage classes, and when should you use them?  
   **A:**  
   - **Standard** – For frequently accessed data.  
   - **Standard-IA (Infrequent Access)** – For data accessed less often but still needs fast retrieval.  
   - **One Zone-IA** – Similar to Standard-IA but stored in a single Availability Zone.  
   - **Glacier** – For long-term archival storage, retrieval time in minutes to hours.  
   - **Glacier Deep Archive** – Cheapest option, retrieval takes hours.  

7️⃣ **Q:** Can you change the storage class of an object after uploading it?  
   **A:** Yes, you can manually change the storage class or use **Lifecycle Policies** to automate the process.  

8️⃣ **Q:** What is S3 Intelligent-Tiering?  
   **A:**  
   - Automatically moves objects between Standard and Infrequent Access based on usage patterns.  
   - Reduces costs without manual intervention.  

---

### **3️⃣ Security & Access Control**  
9️⃣ **Q:** How do you secure objects stored in S3?  
   **A:**  
   - Use **IAM policies** to control user permissions.  
   - Enable **S3 Bucket Policies** to define access rules.  
   - Use **Access Control Lists (ACLs)** for fine-grained permissions.  
   - Enable **S3 Block Public Access** to prevent public exposure.  

🔟 **Q:** What is an S3 bucket policy?  
   **A:** A **JSON-based policy** that defines who can access a bucket and what actions they can perform.  

1️⃣1️⃣ **Q:** How does S3 encryption work?  
   **A:**  
   - **Server-Side Encryption (SSE)**: Encrypts data at rest.  
   - **Client-Side Encryption**: Encrypt data before uploading to S3.  

1️⃣2️⃣ **Q:** What is Versioning in S3?  
   **A:**  
   - Stores multiple versions of an object.  
   - Useful for recovering from accidental deletions.  

1️⃣3️⃣ **Q:** What is S3 Object Lock?  
   **A:**  
   - Prevents objects from being deleted or modified for a specified period (compliance mode).  

---

### **4️⃣ Data Management & Lifecycle**  
1️⃣4️⃣ **Q:** What are S3 Lifecycle Policies?  
   **A:** Rules that automatically move objects between storage classes or delete them after a certain period.  

1️⃣5️⃣ **Q:** What is Cross-Region Replication (CRR)?  
   **A:**  
   - Automatically replicates data to a different AWS region.  
   - Useful for disaster recovery and compliance.  

1️⃣6️⃣ **Q:** What is S3 Transfer Acceleration?  
   **A:**  
   - Uses AWS **Edge Locations (CloudFront)** to speed up uploads.  
   - Reduces latency when transferring data globally.  

---

### **5️⃣ Cost & Pricing**  
1️⃣7️⃣ **Q:** How is S3 pricing calculated?  
   **A:** You pay for:  
   - Storage used (GB/month).  
   - Number of requests (GET, PUT, DELETE).  
   - Data transfer out to the internet.  

1️⃣8️⃣ **Q:** How can you reduce S3 costs?  
   **A:**  
   - Use **Lifecycle Policies** to move data to cheaper storage classes.  
   - Use **Glacier** for archived data.  
   - Enable **Intelligent-Tiering** to optimize costs automatically.  

---

### **6️⃣ Performance & Scalability**  
1️⃣9️⃣ **Q:** How does Amazon S3 scale with large amounts of data?  
   **A:**  
   - Supports unlimited storage.  
   - Can handle **millions of requests per second**.  

2️⃣0️⃣ **Q:** What is the maximum size of an individual object in S3?  
   **A:** **5TB per object**, but the maximum size per upload request is **5GB** (use multipart uploads for larger files).  

2️⃣1️⃣ **Q:** How does S3 handle large file uploads?  
   **A:**  
   - Uses **Multipart Upload**, which splits files into parts and uploads them in parallel.  
   - Improves upload speed and reliability.  

---

### **7️⃣ S3 Event-Driven Architecture**  
2️⃣2️⃣ **Q:** How can S3 trigger actions when files are uploaded?  
   **A:**  
   - Use **S3 Event Notifications** to trigger AWS services like:  
     - **AWS Lambda** (run code when a file is uploaded).  
     - **SNS/SQS** (send notifications or messages).
